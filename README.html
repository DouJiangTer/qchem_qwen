Experiment README

Goal

This viewer compares two LLM ansatz-generation settings on the same 60 stratified PennyLane/qchem tasks. Each task is a molecular VQE-style ansatz task with a benchmark Hamiltonian, FCI energy, Hartree-Fock state, and benchmark VQE information.

The experiment asks whether Qwen can produce or improve Qiskit parameterized ansatz circuits under correctness constraints and E/T/R scoring.

Task Set

The task set contains 60 qchem tasks stratified by molecule, basis, qubit count, and bond length. The run is capped at 12 qubits for statevector-based scoring cost.

Each task prompt asks Qwen to return Python code defining build_ansatz(num_qubits), returning (QuantumCircuit, parameters). The circuit must use the requested number of qubits, include trainable rotations, avoid measurements, and preserve the molecular VQE objective.

Prompt Construction

Reference-start

Round 0 is the benchmark/reference ansatz candidate. The feedback prompt includes:

  • the task prompt;
  • the metric definitions for correctness, E, T, R, and Total;
  • the full round-0 reference Qiskit code;
  • the visible score feedback for the active regime.

Round 2 and later append all previous Qwen candidates for that task, including their visible feedback and code.

From-scratch

The initial generation prompt contains only the task prompt. No reference circuit is shown. Feedback rounds then optimize the Qwen-generated round-0 candidate, again appending all previous Qwen candidates for that task.

Feedback regimes

  • train_only: shows correctness and T only.
  • reliability_only: shows correctness and R only.
  • full_scores: shows correctness, E, T, R, and Total.

Correctness

E/T/R are computed only after correctness passes. A circuit is correct if it:

  • uses exactly the task qubit count;
  • has no classical bits and no measurements;
  • has parameter count within task limits;
  • does not exceed task max depth;
  • uses only allowed gates;
  • has finite Hamiltonian energy at the all-zero parameter vector.
If correctness fails: E = 0, T = 0, R = 0, Total = 0

Expressibility E

The implementation samples N_E = 102 random parameter vectors uniformly from [-pi, pi]^p. It computes all pairwise state fidelities:

F_ij = |<psi(theta_i)|psi(theta_j)>|^2

For 102 samples this gives 5151 fidelities. The fidelities are histogrammed into 50 uniform bins over [0, 1]. The Haar bin mass for dimension d = 2^n and bin [a,b] is:

P_Haar([a,b]) = (1 - a)^(d - 1) - (1 - b)^(d - 1)

Both empirical and Haar histograms use +1e-12 smoothing and normalization. The score is:

D_KL = sum_b P_emp(b) log(P_emp(b) / P_Haar(b))
E = exp(-D_KL)

Trainability T

T is a Hamiltonian-energy gradient-variance barren-plateau proxy. For energy:

L(theta) = <psi(theta)|H|psi(theta)>

The evaluator samples random parameter vectors and checks up to 24 parameters. For parameter index m it uses parameter-shift gradients:

g_{k,m} = 0.5 * (L(theta_k + (pi/2)e_m) - L(theta_k - (pi/2)e_m))

Then:

gradient_variance = Var({g_{k,m}})
logv = log10(gradient_variance + 1e-16)

This run uses strict mapping:

ANSATZ_TRAIN_LOG_MIN = -4
ANSATZ_TRAIN_LOG_MAX = 0
T = clip((logv - (-4)) / (0 - (-4)), 0, 1)
  = clip((log10(gradient_variance + 1e-16) + 4) / 4, 0, 1)

Reliability R

The circuit is transpiled to:

basis_gates = ["rz", "sx", "x", "cx"]
coupling_map = line coupling
optimization_level = 1

Gate counts are taken from the transpiled circuit if transpilation succeeds. One-qubit gates counted are rz, sx, x, z, h, s, sdg, rx, ry, u, u3. Two-qubit gates counted are cx, cz.

p1 = 1e-3
p2 = 1e-2
G1 = one_qubit_gate_count
G2 = two_qubit_gate_count
R = (1 - p1)^G1 * (1 - p2)^G2
  = 0.999^G1 * 0.99^G2

Total

Total = (E + T + R) / 3

For incorrect candidates, Total is zero.
